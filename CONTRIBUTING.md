## How to Contribute

### Reporting Bugs

- Use GitHub Issues
- Include clear description and steps to reproduce
- Provide system information and logs if relevant

### Suggesting Features

- Open a GitHub Issue with [Feature Request] tag
- Describe the use case and expected behavior
- Discuss before starting implementation

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m "Add feature X"`
6. Push to your fork: `git push origin feature/your-feature`
7. Open a Pull Request

### Code Standards

- Follow PEP 8 for Python code
- Add docstrings to functions
- Include tests for new features
- Update documentation as needed

### Testing
```bash
# Run tests
pytest tests/

# Check code style
flake8 app/

# Type checking
mypy app/
```

## Development Setup
```bash
# Clone and setup
git clone https://github.com/Qwerydotxyz/qwery-x402-facilitator.git
cd qwery-x402-facilitator
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-dev.txt

# Run locally
python main.py
```
